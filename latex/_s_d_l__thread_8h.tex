You\textquotesingle{}ll have to see what happens with your system\textquotesingle{}s debugger. The name should be U\+T\+F-\/8 (but using the naming limits of C identifiers is a better bet). There are no requirements for thread naming conventions, so long as the string is null-\/terminated U\+T\+F-\/8, but these guidelines are helpful in choosing a name\+:

\href{http://stackoverflow.com/questions/149932/naming-conventions-for-threads}{\tt http\+://stackoverflow.\+com/questions/149932/naming-\/conventions-\/for-\/threads}

If a system imposes requirements, S\+DL will try to munge the string for it (truncate, etc), but the original string contents will be available from \mbox{\hyperlink{_s_d_l__thread_8h_abe2ffa508d85379b355673d557e74d9f}{S\+D\+L\+\_\+\+Get\+Thread\+Name()}}. \mbox{\Hypertarget{_s_d_l__thread_8h_af90602c51ef5a62f26ed9931af484906}\label{_s_d_l__thread_8h_af90602c51ef5a62f26ed9931af484906}}

\subsubsection{\texorpdfstring{S\+D\+L\+\_\+\+Detach\+Thread()}{SDL\_DetachThread()}}
{\footnotesize\ttfamily \mbox{\hyperlink{begin__code_8h_aa4c7a931f4a968f818b2a1b10a432185}{D\+E\+C\+L\+S\+P\+EC}} \mbox{\hyperlink{_s_d_l__opengles2__gl2ext_8h_ae5d8fa23ad07c48bb609509eae494c95}{void}} \mbox{\hyperlink{begin__code_8h_a81faf4ba0455dc75f2e0507eddb79401}{S\+D\+L\+C\+A\+LL}} S\+D\+L\+\_\+\+Detach\+Thread (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{_s_d_l__thread_8h_a97cd1dc60cea74b74273affb1a974db5}{S\+D\+L\+\_\+\+Thread}} $\ast$}]{thread }\end{DoxyParamCaption})}

A thread may be \char`\"{}detached\char`\"{} to signify that it should not remain until another thread has called \mbox{\hyperlink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{S\+D\+L\+\_\+\+Wait\+Thread()}} on it. Detaching a thread is useful for long-\/running threads that nothing needs to synchronize with or further manage. When a detached thread is done, it simply goes away.

There is no way to recover the return code of a detached thread. If you need this, don\textquotesingle{}t detach the thread and instead use \mbox{\hyperlink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{S\+D\+L\+\_\+\+Wait\+Thread()}}.

Once a thread is detached, you should usually assume the S\+D\+L\+\_\+\+Thread isn\textquotesingle{}t safe to reference again, as it will become invalid immediately upon the detached thread\textquotesingle{}s exit, instead of remaining until someone has called \mbox{\hyperlink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{S\+D\+L\+\_\+\+Wait\+Thread()}} to finally clean it up. As such, don\textquotesingle{}t detach the same thread more than once.

If a thread has already exited when passed to \mbox{\hyperlink{_s_d_l__thread_8h_af90602c51ef5a62f26ed9931af484906}{S\+D\+L\+\_\+\+Detach\+Thread()}}, it will stop waiting for a call to \mbox{\hyperlink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{S\+D\+L\+\_\+\+Wait\+Thread()}} and clean up immediately. It is not safe to detach a thread that might be used with \mbox{\hyperlink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{S\+D\+L\+\_\+\+Wait\+Thread()}}.

You may not call \mbox{\hyperlink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{S\+D\+L\+\_\+\+Wait\+Thread()}} on a thread that has been detached. Use either that function or this one, but not both, or behavior is undefined.

\subsubsection{\texorpdfstring{S\+D\+L\+\_\+\+Wait\+Thread()}{SDL\_WaitThread()}}
{\footnotesize\ttfamily \mbox{\hyperlink{begin__code_8h_aa4c7a931f4a968f818b2a1b10a432185}{D\+E\+C\+L\+S\+P\+EC}} \mbox{\hyperlink{_s_d_l__opengles2__gl2ext_8h_ae5d8fa23ad07c48bb609509eae494c95}{void}} \mbox{\hyperlink{begin__code_8h_a81faf4ba0455dc75f2e0507eddb79401}{S\+D\+L\+C\+A\+LL}} S\+D\+L\+\_\+\+Wait\+Thread (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{_s_d_l__thread_8h_a97cd1dc60cea74b74273affb1a974db5}{S\+D\+L\+\_\+\+Thread}} $\ast$}]{thread,  }\item[{int $\ast$}]{status }\end{DoxyParamCaption})}

Wait for a thread to finish. Threads that haven\textquotesingle{}t been detached will remain (as a \char`\"{}zombie\char`\"{}) until this function cleans them up. Not doing so is a resource leak.

Once a thread has been cleaned up through this function, the S\+D\+L\+\_\+\+Thread that references it becomes invalid and should not be referenced again. As such, only one thread may call \mbox{\hyperlink{_s_d_l__thread_8h_ac96743626ed21d2cdb0868a23cca9705}{S\+D\+L\+\_\+\+Wait\+Thread()}} on another.

The return code for the thread function is placed in the area pointed to by {\ttfamily status}, if {\ttfamily status} is not N\+U\+LL.

You may not wait on a thread that has been used in a call to \mbox{\hyperlink{_s_d_l__thread_8h_af90602c51ef5a62f26ed9931af484906}{S\+D\+L\+\_\+\+Detach\+Thread()}}. Use either that function or this one, but not both, or behavior is undefined.

It is safe to pass N\+U\+LL to this function; it is a no-\/op.
