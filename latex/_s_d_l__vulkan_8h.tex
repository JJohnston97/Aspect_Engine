If {\itshape p\+Names} is {\ttfamily N\+U\+LL}, then the number of required Vulkan instance extensions is returned in p\+Count. Otherwise, {\itshape p\+Count} must point to a variable set to the number of elements in the {\itshape p\+Names} array, and on return the variable is overwritten with the number of names actually written to {\itshape p\+Names}. If {\itshape p\+Count} is less than the number of required extensions, at most {\itshape p\+Count} structures will be written. If {\itshape p\+Count} is smaller than the number of required extensions, {\ttfamily S\+D\+L\+\_\+\+F\+A\+L\+SE} will be returned instead of {\ttfamily S\+D\+L\+\_\+\+T\+R\+UE}, to indicate that not all the required extensions were returned.

\begin{DoxyNote}{Note}

The extension names queried here must be enabled when calling Vk\+Create\+Instance, otherwise surface creation will fail.

{\ttfamily window} should have been created with the {\ttfamily S\+D\+L\+\_\+\+W\+I\+N\+D\+O\+W\+\_\+\+V\+U\+L\+K\+AN} flag.
\end{DoxyNote}

\subsubsection{\texorpdfstring{S\+D\+L\+\_\+\+Vulkan\+\_\+\+Load\+Library()}{SDL\_Vulkan\_LoadLibrary()}}
{\footnotesize\ttfamily \mbox{\hyperlink{begin__code_8h_aa4c7a931f4a968f818b2a1b10a432185}{D\+E\+C\+L\+S\+P\+EC}} int \mbox{\hyperlink{begin__code_8h_a81faf4ba0455dc75f2e0507eddb79401}{S\+D\+L\+C\+A\+LL}} S\+D\+L\+\_\+\+Vulkan\+\_\+\+Load\+Library (\begin{DoxyParamCaption}\item[{const char $\ast$}]{path }\end{DoxyParamCaption})}



Dynamically load a Vulkan loader library. 


\begin{DoxyParams}[1]{Parameters}
\mbox{\tt in}  & {\em path} & The platform dependent Vulkan loader library name, or {\ttfamily N\+U\+LL}.\\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
{\ttfamily 0} on success, or {\ttfamily -\/1} if the library couldn\textquotesingle{}t be loaded.
\end{DoxyReturn}
If {\itshape path} is N\+U\+LL S\+DL will use the value of the environment variable {\ttfamily S\+D\+L\+\_\+\+V\+U\+L\+K\+A\+N\+\_\+\+L\+I\+B\+R\+A\+RY}, if set, otherwise it loads the default Vulkan loader library.

This should be called after initializing the video driver, but before creating any Vulkan windows. If no Vulkan loader library is loaded, the default library will be loaded upon creation of the first Vulkan window.

\begin{DoxyNote}{Note}
It is fairly common for Vulkan applications to link with {\itshape libvulkan} instead of explicitly loading it at run time. This will work with S\+DL provided the application links to a dynamic library and both it and S\+DL use the same search path.

If you specify a non-\/\+N\+U\+LL {\ttfamily path}, an application should retrieve all of the Vulkan functions it uses from the dynamic library using {\ttfamily \mbox{\hyperlink{_s_d_l__vulkan_8h_aee3a196487ad045f8c2b5a0708f5bf55}{S\+D\+L\+\_\+\+Vulkan\+\_\+\+Get\+Vk\+Get\+Instance\+Proc\+Addr()}}} unless you can guarantee {\ttfamily path} points to the same vulkan loader library the application linked to.

On Apple devices, if {\itshape path} is N\+U\+LL, S\+DL will attempt to find the vk\+Get\+Instance\+Proc\+Addr address within all the mach-\/o images of the current process. This is because it is fairly common for Vulkan applications to link with libvulkan (and historically Molten\+VK was provided as a static library). If it is not found then, on mac\+OS, S\+DL will attempt to load {\ttfamily vulkan.\+framework/vulkan}, {\ttfamily libvulkan.\+1.\+dylib}, {\ttfamily Molten\+V\+K.\+framework/\+Molten\+VK} and {\ttfamily lib\+Molten\+V\+K.\+dylib} in that order. On i\+OS S\+DL will attempt to load {\ttfamily lib\+Molten\+V\+K.\+dylib}. Applications using a dynamic framework or .dylib must ensure it is included in its application bundle.

On non-\/\+Apple devices, application linking with a static libvulkan is not supported. Either do not link to the Vulkan loader or link to a dynamic library version.

This function will fail if there are no working Vulkan drivers installed.
\end{DoxyNote}
